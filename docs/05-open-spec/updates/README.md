# Updates Open-Spec

Aquí viven los **SPEC-update** cuando el alcance cambia en control de calidad o refinamiento, en la **misma jerarquía** que las historias y tareas:

- `updates/000-Generalidades/`
- `updates/001-Seguridad/`

Convención de nombres: `SPEC-NNN-slug-update.md`, `SPEC-NNN-slug-update-01.md`, … (ver `guia-adopcion-open-spec.md` y `08-open-spec-gobernanza`).

Las carpetas se crean al primer update; no es obligatorio tener archivos aquí solo por existir los SPEC base.
